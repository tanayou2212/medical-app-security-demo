# medical-app-security-demo

## Purpose

Demonstration project for medical device software security vulnerabilities and mitigations.
Used for security education, regulatory compliance training (FDA, EU MDR, MHLW), and CTF-style exercises.

## Directory Structure

- `vulnerable/` — Intentionally vulnerable medical app code (for analysis and education)
- `attack/` — Proof-of-concept exploits and attack scripts (authorized testing only)
- `secure/` — Hardened / remediated versions of the vulnerable code
- `docs/` — Security analysis reports, threat models, and compliance notes

## Rules

- All code comments must be written in English
- All README files must be written in English
- Response in Japanese is acceptable for conversation
- Code in `attack/` is for authorized security testing and educational purposes only
- Do not include real patient data or credentials anywhere in this repository
- Regulatory references (FDA 21 CFR Part 11, EU MDR Annex I, MHLW guidelines) must be cited accurately

## Regulatory Context

| Framework | Scope |
|-----------|-------|
| FDA 21 CFR Part 11 | Electronic records / electronic signatures |
| FDA Cybersecurity Guidance (2023) | Pre/post-market cybersecurity for medical devices |
| EU MDR 2017/745 Annex I | General safety and performance requirements |
| MHLW Notification No. 0323-1 (2021) | Cybersecurity guidance for medical devices (Japan) |
